# TESTING RULES — KJSIS

## TESTING FRAMEWORK

- Use Jest for backend testing

## TEST COVERAGE

Must include:
- Controllers
- Services
- Utility functions

## TEST TYPES

### SUCCESS CASES
- Valid inputs
- Expected outputs

### FAILURE CASES
- Invalid inputs
- Unauthorized access
- Missing data

### EDGE CASES
- Empty datasets
- Boundary values
- Large datasets

## SPECIAL TESTING RULES

- Test role-based access control
- Test mark submission flow
- Test attendance logic (default present)
- Test exam locking behavior

## GOAL

Ensure system reliability for daily school operations
