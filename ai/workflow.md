# AI DEVELOPMENT WORKFLOW — KJSIS

When implementing any feature, always follow this order:

## ANALYZE
- Understand feature requirement
- Identify affected modules

## DESIGN
- Plan database changes
- Plan API endpoints
- Plan data flow

## IMPLEMENT
- Write backend services first
- Then controllers and routes
- Then frontend

## VALIDATE
- Ensure logic matches real school workflow
- Check role-based access

## TEST
- Test success scenarios
- Test failure scenarios
- Test edge cases

## REVIEW
- Check code quality
- Check security
- Check performance

## IMPROVE
- Suggest optimizations
- Suggest UI improvements

## IMPORTANT RULES

- Never break existing features
- Maintain backward compatibility
- Prefer simple solutions over complex ones
